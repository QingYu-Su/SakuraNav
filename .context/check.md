# SakuraNav - AI 提交前检查清单

> 在执行 `git commit` 前，AI 必须按照本清单逐项检查所有修改的代码。

## 检查项

### 结构与规范

- [ ] 是否遵循 `.context/develop.md` 中的全部规范
- [ ] 单文件是否超过 1000 行？超过则需拆分
- [ ] 新增文件是否放到了职责对应的目录？是否存在文件与子目录混放
- [ ] 是否绕过了项目已有的架构模式（Repository、Zod、types.ts 等）

### 代码质量

- [ ] 代码是否具有鲁棒性（边界处理、异常捕获、空值防护）
- [ ] 代码是否足够优雅（逻辑清晰、无冗余、命名合理）
- [ ] 关键逻辑是否有注释
- [ ] 重要功能是否添加了日志

### 功能与 UI

- [ ] 改动是否专注，未影响无关功能
- [ ] UI 改动是否在明亮/暗黑/壁纸/移动端下均适配
- [ ] 是否引入了 `any` 类型

### 验证

- [ ] `npm run lint` 是否零错误
- [ ] `npm run build` 是否零错误

## 评分

对本次修改整体打分（0-10 分）：

| 维度 | 分数 (0-10) |
|:-----|:------------|
| 规范遵循 | |
| 代码质量 | |
| 改动专注度 | |
| 鲁棒性 | |
| **总分** | **/ 40（换算为 10 分制）** |

> 换算公式：总分 ÷ 4 = 最终评分
>
> **最终评分 ≥ 8 分** → 执行以下操作：
> 1. 根据本次代码变更，同步更新 `docs/` 下的开发文档（DEVELOPMENT.md、DOCKER.md 等），**不包含 README.md 、README_EN.md 和 CHANGELOG.md**
> 2. 建议提交 commit
>
> **最终评分 < 8 分** → 列出需要修复的问题，修复后重新检查
