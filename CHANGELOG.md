# 更新日志

## [1.1.5] - 2026-04-16

### ✨ 新功能

- **AI 智能分析** - 编辑网站时新增「AI 分析」按钮，自动识别网站标题、描述、关联标签和推荐新标签，支持配置自定义 AI 模型
- **网站编辑器内嵌标签管理** - 编辑网站时可直接新建/编辑标签，无需跳转到标签管理页面

### 🎨 UI 优化

- **标签栏精简** - 移除标签卡片上的图标显示，标签编辑仅需名称和描述，侧边栏更窄更简洁
- **标签栏居中排版** - 标签卡片标题和描述居中对齐，支持自动换行，编辑模式下侧边栏自动延展显示编辑按钮
- **拖拽手柄优化** - 标签卡片和网站卡片的拖拽按钮替换为顶部细横线，更轻量优雅
- **网站卡片图标强制正方形** - 图标容器使用固定尺寸 + overflow-hidden，确保所有图标均为正方形
- **编辑按钮样式优化** - 光明模式下网站卡片和标签卡片的编辑按钮颜色加深，提升可见性

### 🐛 Bug 修复

- **修复网站描述为空时创建失败** - 数据库 `description` 列移除 NOT NULL 约束，支持无描述创建网站
- **修复编辑按钮触发页面跳转** - 点击网站卡片编辑按钮时不再冒泡触发 `<a>` 标签导航
- **修复拖拽时卡片尺寸变化** - 拖拽 overlay 卡片锁定原始宽高，无标签网站拖拽时不再缩小
- **修复官方图标获取失败仍显示** - favicon.im 返回 1x1 空白图或 404 时不显示「官方图标」选项
- **修复 AI 分析后图标选择逻辑** - AI 分析完成后优先选择官方图标，不可用时自动选择文字图标
- **修复站点名称修改不生效** - 修改站点名称后浏览器标签标题实时更新，并增加防抖保存

## [1.1.0] - 2026-04-16

### ✨ 新功能

- **站点名称自定义** - 在「其他」配置面板中新增站点名称设置，支持最多 30 字符，自动保存，实时更新浏览器标签标题和导航栏显示名称
- **网站卡片图标背景色** - 编辑网站时可为图标设置自定义背景色，支持文字图标和官方图标的背景色定制
- **文字图标** - 编辑网站时新增「文字图标」选项，自动取网站名首字生成图标，支持自定义背景色
- **标签 Logo 自定义** - 标签栏支持为每个分类标签设置独立的 Logo 图标
- **网站编辑器图标交互重构** - 图标选择改为可视化选项卡模式，支持「当前图标 / 文字图标 / 官方图标 / 上传图标」快速切换
- **标签切换过渡动画** - 切换标签时新增类 pjax 过渡效果（缩放 + 模糊 + 透明度），顶部进度条 sweep 动画，新内容卡片依次入场
- **官方图标动态加载** - 选择「官方图标」时不再下载到服务器本地，直接使用 favicon.im 动态 URL，节省服务器存储空间且图标始终为最新
- **图标加载失败 Fallback** - 动态加载的官方图标加载失败时自动降级显示网站首字母

### 🐛 Bug 修复

- **修复 npm install 报错** - 修正 package.json 依赖配置，解决安装失败问题
- **修复 Docker 读取配置文件失败** - 修正 docker-entrypoint.sh 中配置文件路径和值解析逻辑

### 🔧 改进

- **标签取消收起逻辑** - 移除分类标签的收起/折叠行为，标签始终完整展示
- **网站编辑器 UI 优化** - 图标编辑区域重新设计，操作流程更直观

## [1.0.5] - 2025-04-02

### 🐛 Bug 修复

- **修复配置文件修改后不生效的问题** - 重启项目后 `admin.path`、`admin.username`、`admin.password` 等配置无法生效（`server-config.ts`）
- **修复自定义登录路径 404 的问题** - 当 `path` 值为纯数字时（如 `5201314`），YAML 会将其解析为数字类型，导致与字符串比较失败（`server-config.ts`）
- **修复构建失败时日志被截断的问题** - `build-and-run.js` 在 lint 或 build 出错时只输出部分错误信息，甚至不输出原因
- **修复 Ctrl+C 强制退出后 `.next` 目录被锁定的问题** - 终止服务后 `.next/standalone` 目录仍被 Node 进程占用，导致重新构建时报 `EBUSY` 错误（`build-and-run.js`）

### 🔧 改进

- **配置文件格式优化** - `config.example.yml` 中用户名、密码、登录路径等参数使用双引号包裹，避免 YAML 类型推断问题
- **Docker 入口脚本改进** - `docker-entrypoint.sh` 正确提取双引号包裹的配置值，新增 `path` 字段支持
- **动态路由强制刷新** - 为 `[...slug]` 路由添加 `force-dynamic`，确保配置变更实时生效（`src/app/[...slug]/page.tsx`）
- **构建阶段自动禁用日志** - 构建过程中自动关闭日志文件输出，避免构建报错（`src/lib/logger.ts`）

### 📝 文档

- 更新中文 README 和英文 README 中的配置示例，使用双引号包裹配置值

## [1.0.0] - 2025-04-01

### 🎉 首个正式版本发布

SakuraNav 是一个基于 Next.js + React + TypeScript + SQLite 的优雅个人导航页。

### 界面体验

- **响应式设计** - 完美适配桌面端和移动端，自适应布局
- **明暗主题** - 支持明亮 / 暗黑主题切换，支持跟随系统
- **主题定制** - 明暗主题分别配置壁纸、字体、透明度、文字颜色
- **动态背景** - 明亮模式飘落樱花花瓣（80 个），暗黑模式闪烁星星（30 颗）
- **毛玻璃卡片** - 桌面端和移动端可独立开关卡片毛玻璃效果
- **桌面/移动壁纸分离** - 桌面端和移动端可分别设置不同壁纸
- **三种字体预设** - Space Grotesk（科技感）、Noto Serif SC（杂志感）、Noto Sans SC（日常清晰）
- **Logo / Favicon 自定义** - 支持上传自定义 Logo 和 Favicon
- **渐进式加载** - 默认"全部网站"模式和标签视图下均支持无限滚动分页加载（每页 12 条）

### 登录管理

- **隐藏登录路由** - 自定义登录入口路径（`admin.path`），游客无法发现管理入口
- **单用户登录** - 支持 30 天免登录，记住登录状态
- **管理员二次确认** - 执行敏感操作时需输入密码确认
- **权限控制** - 登录后可见隐藏标签与隐藏站点
- **编辑器控制台** - 独立的 `/editor` 管理后台页面

### 标签管理

- **标签分类** - 网站按标签分类展示，支持标签 Logo 设置
- **拖拽排序** - 登录后支持拖拽调整标签顺序
- **隐藏标签** - 可设置隐藏标签，仅登录后可见
- **侧边栏折叠** - 移动端侧边栏自动折叠

### 网站管理

- **网站增删改** - 登录后可管理网站信息
- **拖拽排序** - 登录后在单个标签下拖拽网站顺序
- **网站置顶** - 支持网站置顶显示（`is_pinned`）
- **批量管理** - 支持网站与多个标签关联
- **批量编辑** - 编辑器控制台支持批量管理所有标签和网站

### 搜索功能

- **多引擎支持** - Google / Baidu / 站内搜索切换
- **站内搜索** - 在当前视图范围内搜索站点名、描述、标签
- **搜索建议** - 浮动搜索弹窗，支持实时搜索建议
- **键盘导航** - 搜索建议支持上下方向键选择、Enter 确认、Escape 关闭

### 数据管理

- **壁纸上传** - 本地上传壁纸，或通过 URL 下载
- **配置导入导出** - 支持配置数据的导入和导出（ZIP 格式，含数据库与资源文件）
- **配置重置** - 支持一键重置到默认配置
- **示例数据** - 首次运行自动创建示例标签、网站和默认主题配置

### 技术特性

- **全栈 TypeScript** - 前后端统一类型安全
- **React 19** - 使用最新 React 特性（useEffectEvent、useDeferredValue、useTransition）
- **React Compiler** - 启用 React Compiler 自动优化渲染性能
- **Repository Pattern** - 数据访问层采用 Repository 模式封装
- **自动数据库迁移** - SQLite 表结构变化自动检测和迁移
- **WAL 模式** - SQLite 启用 Write-Ahead Logging 提升并发性能
- **JWT 认证** - 基于 jose 库的 JWT + HttpOnly Cookie 安全认证
- **Server-Only Config** - 敏感配置使用 server-only 保护，仅在服务端可访问
- **日志系统** - 自定义日志记录器，支持文件输出和 30 天自动保留
- **Docker 部署** - 多阶段构建，非 root 用户运行，standalone 输出模式
- **YAML 配置** - 外部 YAML 配置文件，支持自定义端口和管理员凭据
